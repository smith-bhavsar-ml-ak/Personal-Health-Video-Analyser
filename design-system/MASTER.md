## 14. Adaptive Theming (Flutter)

The Flutter app follows **system theme** (`ThemeMode.system`). Do NOT hardcode dark-mode surfaces.

### colorScheme.* substitution table
| Old hardcoded AppColors | Use instead |
|-------------------------|-------------|
| `AppColors.surface` / `AppColors.surfaceDark` | `cs.surface` |
| `AppColors.border` | `cs.outline` |
| `AppColors.surfaceElevated` | `cs.surfaceContainerHigh` |
| `AppColors.textPrimary` | `cs.onSurface` |
| `AppColors.textSecondary` / `AppColors.textMuted` | `cs.onSurfaceVariant` |

### Brand colors (safe to use directly)
`AppColors.primary` (#6366F1), `AppColors.health` (#10B981), `AppColors.warning` (#F59E0B), `AppColors.error` (#EF4444), `AppColors.info` (#38BDF8) ‚Äî these are intentional brand colors and do NOT need to be replaced by colorScheme tokens.

## 18. Navigation Architecture

### Research basis
Follows Strava / Strong pattern: **5 direct tabs, no "More" button**, ordered by daily-use frequency.
- 3‚Äì5 tabs is Material Design + HIG consensus
- Scrollable tab bars are an anti-pattern (users don't expect to scroll)
- Hamburger menus reduce discoverability; only use for overflow > 5 items

### Mobile ‚Äî Bottom Tab Bar (5 items)
| # | Destination | Why primary |
|---|-------------|-------------|
| 1 | **Home** | Daily dashboard: streak, steps, recent sessions |
| 2 | **Analyze** | Core value prop ‚Äî video upload; gets subtle indigo tint when inactive |
| 3 | **Activity** | Daily steps/walking ‚Äî habit loop, opened every day |
| 4 | **Plans** | Today's workout plan ‚Äî daily check-in |
| 5 | **Progress** | Charts/analytics ‚Äî frequent but secondary to actions |

Active tab indicator: 3px indigo dot above the icon (Strava-style), not a full bg highlight.

Secondary screens reachable within primary tabs:
- **History** ‚Üí Home "View All sessions" ‚Üí `/history`
- **Library** ‚Üí Plans screen ‚Üí `/library`
- **AI Coach** ‚Üí Session detail mic button ‚Üí `/assistant`
- **Settings** ‚Üí Profile icon in app bar ‚Üí `/settings`
- **Paywall** ‚Üí Any gated feature ‚Üí `/paywall?hint=...`

## 19. Anti-Patterns (Never Do)

- No gradient backgrounds on page or cards
- No border-radius > 16px on data cards
- No emoji as icons ‚Äî use Lucide SVG
- No pure black (#000000) backgrounds ‚Äî use #0A0A0F
- No white text on colored backgrounds (use brand-colored text on dimmed bg)
- No hover states that cause layout shift
- No fixed px font sizes below 14px
- No color as the only indicator of state (add icon or text)
